Networkmanager 启动

  • 在公司实习的时候碰到一个问题，在无线使用的时候之前都是直接用wifi-menu来扫描无线，然后直接连接即可；然而到了公司发现原来他们把无线给隐藏起来了，那我这种方法就完全不能使用了，不得不装带有GUI的网络工具，一般有两个选择，NetworkManager和Wicd，开始的时候装了wicd，但是死活无法连上隐藏的无线，不得已又装了NetworkManager，这个就怪了，装了之后只要用systemctl enable Networkmanager启动之后就导致无法联网，看了好久的wiki才发现原来dhcpcd和NetworkManager存在一些冲突，用命令systemctl disable dhcpcd.service之后重启电脑就搞定了，界面工具的话装了一个nm-applet，然后我用的是slim启动X，所以在~/.xinitrc中添加 nm-applet –sm-disable &,这样的话在X启动的时候就会直接启动界面网络管理工具；

goagent Crypto.Cipher.ARC4 Failed

  • 在root权限下运行命令python2.7 proxy.py出现以下提示：

  • 解决办法：
    goagent会尝试调用python内置的Crypto.Cipher.ARC4进行RC4加密/解密，它属于pycrypto软件包。如果缺少这个软件包，则会产生一个调用失败的错误，这时goagent会使用自带的RC4加解密模块代替。后者在功能上与前者完全一致，在性能上略有损失，但在网速不是很快的时候几乎不会感觉到有差别。如果你的网速超过10Mbps或1MB/s，那么可能会体现为CPU占用率有所上升。直接安装 emerge dev-python/pycrypto 就可以解决

awesome-wm eclipse 闪退

最近要用到java，于是便安装了eclipse，安装的时候出现各种奇葩问题，其中最让我无语的就是在 awesome-wm 的全屏布局下完全没有问题，但是在其他桌面布局下出现莫名的闪退，而且还直接将同一桌面下的所有终端给关闭，闪退后因为终端也别强制关闭，所以看不到任何的输出提示，尝试了半天发现有一种方法有效，解决方法如下：

  1. 安装wmname
    emerge -q wmname
  2. 命名window manager
    wmname LG3D
    至于原因awesome的wiki上面也有提到：
  • The X Window System integration in most modern implementations of the Java virtual machine (JVM) displays only empty, gray windows when used with many non-re-parenting window managers, such as Awesome. This is almost certainly a violation of ICCCM and has been reported to Sun Microsystems a number of times.

awesome+gentoo+dolphin无法挂载u盘

dolphin下可以显示盘符，但是无法自动挂载，挂载是提示：

经过一番的google发现产生错误的原因的三点：

  1. 现在用的用户没有在 storage 用户组，但是升级之前都 OK 的，这个应该不是关键
  2. /usr/share/polkit-1/actions/org.freedesktop.udisks.policy 这个文件但是帖子里面没有给出有用的线索为什么是它惹的祸，我也没动它
  3. 提到 ck-launch-session 这个，slim 和 ~/.xinitrc 文件都可以定义启动桌面管理器 session 前，启用这个这个进程，fcitx 也会依赖它
    原来 slim 后来的版本默认已经内置启用了 ConsoleKit 不需要再手动在 slim.conf 或 ~/.xinitrc 中制定 ck-launch-session 了slim archwiki 中也在 NOTE 中标记了这个变化
    之后在 ~/.xinitrc 中去掉 ck-launch-session 后，启动后 dolphin 可以自动挂载了

gentoo中mysql配置

在配置mysql时，如果按照gentoo wiki 上配置的话会发现找不到mysql_install_db命令，其实 mysql_install_db是作为一个脚本放在系统文件夹中的，具体配置步骤如下:

  1. 执行 /usr/share/mysql/scripts/mysql_install_db --basedir=/usr
  2. /etc/init.d/mysql start
  3. /usr/bin/mysql_secure_installation
  4. mysql -u root -p -h localhost
