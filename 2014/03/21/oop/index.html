基本概念

  1. 类(class): 具有相似的内部状态和运动规律的实体集合。
  2. 对象(object): 指现实世界中各种各样的实体，也就是类(class)的实例。
  3. 消息(message): 指对象间相互联系和相互作用的方式。
  4. 类的特性: 抽象、继承、封装、重载、多态

class和struct的区别

  • 在c语言中： struct只能作为一种复杂的数据类型定义，不能用于面向对象编程
  • 在c++中，对于成员访问权限以及继承方式，class中默认的是private，而struct中则是public.

c++类的四个默认函数

在我们定义一个类之后，就算我们不做任何的初始化，c++也会自动为我们生成以下几个函数：

  1. 默认构造函数
    默认构造函数是只不带有任何参数或者所有参数均为缺省值的构造函数，是一种特殊的成员函数，无返回值，函数名与类同名。它提供了对成员变量进行初始化的方法，使得在声明对象时能自动地初始化对象。
  2. 拷贝构造函数
    拷贝构造函数是c++类中提供的一个默认的来处理复制的函数，一般而言我们没必要重新定义，拷贝构造函数是在对象被创建时调用的
  3. 赋值构造函数
    赋值构造函数与拷贝构造函数的区别：拷贝构造函数是对象被创建时调用，赋值函数只能被已经存在了的对象调用
  4. 析构函数
    析构函数也是特殊的类成员函数，它没有返回类型，没有参数，只有在类对象的生命期结束的时候，由系统自动调用，用来在系统释放对象前做一些清理工作。定义析构函数因使用”~”符号加类名(逻辑非运算符)，表示它为逆构造函数，它不能带任何参数

static类成员

非static数据成员一般存在于类类型的每个对象中，然而static数据成员则是直接与类关联，而不是与类的对象关联

  • tips：
    1. static成员的名字是在类的作用域中，因此可以直接用类名访问static成员
    2. 可以实施封装，static成员可以是私有的，而全局对象不可以
    3. 更容易清晰显示出成员变量与类的关联
    4. static类成员函数因为不是任何类对象的组成部分，所以static成员函数没有this指针
    5. static数据成员不是通过构造函数初始化，而是在定义时就已经初始化

多态

多态属于面向对象另一个特征，多态性可以简单地概括为“一个接口，多种方法”，程序在运行时才决定调用的函数，它是面向对象编程领域的核心概念。多态(polymorphisn)，字面意思多种形状。

C++多态性是通过虚函数来实现的，虚函数允许子类重新定义成员函数，而子类重新定义父类的做法称为覆盖，或者称为重写。而重载则是允许有多个同名的函数，而这些函数的参数列表不同，允许参数个数不同，参数类型不同，或者两者都不同。编译器会根据这些函数的不同列表，将同名的函数的名称做修饰，从而生成一些不同名称的预处理函数，来实现同名函数调用时的重载问题。但这并没有体现多态性。

  • 覆盖

这是一个典型的覆盖问题，Parent类里的foo函数是一个虚函数，虚函数是会子类同名函数所覆盖的，而Parent类里的foo1函数是一个普通的函数，不会被子类同名函数所覆盖，p是一个Parent类型的指针，但指向的是Son类型的对象，当调用的是虚函数时就自动调用了指向对象的函数，否则就输出父类对象的函数。

继承

继承是面向对象的一个基本特征，是c++一个非常重要的机制，继承特性可以使一个子类获得父类的操作和数据，并依据自己的需要添加自己要实现的部分。

公有继承

继承成员保持自己的访问级别，基类的public成员称为派生类的public成员，基类的protected成员称为派生类的protected的派生成员，子类可以访问基类的公有和保护成员，只能修改基类的公有成员

保护继承

基类的public和protected成员在派生类中为protected成员，只能访问，不能修改

私有继承

基类的所有成员称为派生类的私有成员，原则上，C++类中私有变量不允许在类、friend类之外的其他任何地方访问,即使是在类外直接用类名访问该类的私有成员也是不允许的，若想真的在类外访问，一般借助指针来实现。

继承层次

理解c++中的继承层次关键在于理解如何确定函数的调用，一般调用遵循以下四个步骤：

  1. 首先确定进行函数调用的对象、引用或指针的静态类型
  2. 在该类中查找函数，如果找不到，就在直接基类中查找，如此遵循着类的继承往上找，直到找到该函数或者查找完最后一个类，如果不能在类或其相关基类中找到该名字，则调用是错误的
  3. 一旦找到该名字，就进行常规类型检查，查看如果给定找到的定义，该函数调用是否函数
  4. 假定函数调用合法，编译器就生成代码，如果函数是虚函数且通过引用或指针调用，则编译器生成代码以确定根据对象的动态类型运行哪个函数版本，否则，编译器生成代码直接调用函数

纯虚函数

定义

纯虚函数是在基类中声明的虚函数，它在基类中没有定义，但要求任何派生类都要定义自己的实现方法。在基类中实现纯虚函数的方法是在函数原型后加“=0”
virtual void funtion()=0

引入原因

  1. 为了方便使用多态特性，我们常常需要在基类中定义虚拟函数。
  2. 在很多情况下，基类本身生成对象是不合情理的。例如，动物作为一个基类可以派生出老虎、孔雀等子类，但动物本身生成对象明显不合常理。为了解决上述问题，引入了纯虚函数的概念，将函数定义为纯虚函数（方法：virtual ReturnType Function()= 0;），则编译器要求在派生类中必须予以重写以实现多态性。
