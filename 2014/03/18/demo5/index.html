指针和引用的区别

  1. 非空区别
    在任何情况下都不能使用指向空值的引用，即引用必须初始化，必须指向某些对象;当声明一个不指向任何对象的变量时最好使用指针;
  2. 合法性区别
    在使用引用之前不需要测试它的合法性，相反，指针则应该总是被测试，防止为空;
  3. 可修改的区别
    指针可以被重新赋值，但引用必须且一直指向在初始化时初始化的对象;

不同的指针

  1. 函数指针
    void (*f)();虽然()的优先级是高于*的，但是由于括号的存在所以f首先是一个指针，接下来执行( )，表明f指向一个函数，这个函数不返回任何值。因此f是一个指向不接受任何参数的且没有返回值的函数的指针，简称函数指针;
  2. 指针函数
    int* f();f首先与括号结合，所以f是一个函数，不接受任何参数，返回值是一个int型的指针;
  3. const指针
    int * const p : p是指向int型的指针，但指针本身为const型，即指针的地址不会变，指向内容可以改变，即(*p)++是可以的，但p++不允许
  4. 指向const的指针
    const int *p;指针本身的地址可以改变，但是指针指向的变量为const int类型，即所指向的变量不能改变;等价与int const *p,因此(*p)++是不允许的，但是p++是可以的
  5. 指向const的const指针
    const int* const p;指针本身不能改变，同时它所指向的变量也不能发生改变，(*p)++和p++都是不允许的
  6. this指针
    • this指针只能在成员函数中使用 全局函数、静态函数都不能使用this
    • this 生命周期 this指针在成员函数开始前构造，并在成员的结束后清除
  7. int(*(*F)(int,int))(int)剖析
    由于括号的存在，首先F是一个指针*F，由于括号的优先级高于*，于是先于后边的结合，所以F是一个函数指针，该函数带有两个int型的参数，再与前边的*结合，说明该函数返回一个指针，这个指针指向的是一个带有int型参数且返回int型数据的函数

  8. 迷途指针
    c中的野指针，c++的迷途指针都是不是NULL的指针，也不是指向常量的指针，而是指向“垃圾”内存的指针;

    • 指针变量没有被初始化。任何指针变量刚被创建时不会自动成为NULL指针，它的缺省值是随机的，它会乱指一气。所以，指针变量在创建的同时应当被初始化，要么将指针设置为NULL，要么让它指向合法的内存。

    • 当我们没有将指针重新赋值就再试图再次使用该指针，引起的结果是不可预料的，因此我们在将指针删除之后就不要再使用它，虽然这个指针仍然指向原来的内存区域，但是编译器已经把该内存重新分配给了其他的数据，再次使用很有可能导致你的程序崩溃。

指针与数组

  • 指向指针的指针
    指针可以指向基本类型，也可以指向复合类型，因此也可以指向另外一个指针变量，称为指向指针的指针。

  • 数组指针
    数组中的每个元素可以是基本类型，也可以复合类型，因此也可以是指针类型。例如定义一个数组a由10个元素组成，每个元素都是int *指针：

  • 指向数组的指针
    指针可以指向复合类型，因此必然可以指向数组类型;以下定义一个指向数组的指针：
    int (*p)[];
    解释方法与函数指针一样，[]的优先级高于*，但由于括号的存在，p首先是一个指针，然后说明是一个数组，数组中数据类型为int型，所以p是一个指向int型数组的指针；

    • 使用

a是一个数组，在&a这个表达式中，数组名做左值，取整个数组的首地址赋给指针p。注意，&a[0]表示数组a的首元素的首地址，而&a表示数组a的首地址，显然这两个地址的数值相同，但这两个表达式的类型是两种不同的指针类型，前者的类型是int *，而后者的类型是int (*)[10]。

  • 指针与二维数组
    由于我是做图像处理的，因此难免要用到二维数组，所以个人觉得二维数组跟指针的关系很有必要弄清楚;

这时p[0]的值与a[0]的值是一样的，取的都是同一个元素，因为此时a[0]本身也是一个指针，此时里面存储的是10个int型元素，即a[0]是由10个int组成的数组，而不是基本类型，这样我们可以把p当做一个二维数组来使用，pa[1][2]和a[1][2]取的也是同一个元素，而且pa比a用起来更灵活，数组名(数组名本身即为一个指针)不支持赋值、自增等运算，而指针可以支持，p++使p跳过二维数组的一行（40个字节），指向a[1]的首地址。
