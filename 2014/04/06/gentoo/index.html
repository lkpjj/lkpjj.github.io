正式转向gentoo

  • linux
linux

最近几天一直在安装gentoo，之前一直使用的archlinux，arch没有什么不好，只是它更新的太过勤快，所以极偶尔会出现一点点的小问题，于是便萌生了再次折腾gentoo的想法(之前一次以失败告终);

个人认为gentoo和archlinux相比有几个优点：

  1. 定制性更加出众，当然这也意味着安装起来更加困难
  2. 包管理更加方便，可以依据自己的需求安装稳定或者不稳定的软件版本
  3. 所有的软件都必须自己重新编译，这样可以对自己的系统做出更好的优化
  4. 让你自己动手编译内核，这也让你更多的了解linux
  5. Gentoo主要是基于源码的而 Arch 主要是基于二进制的。两个都是滚动式平滑更新的发行版
  6. 伟大的USE标记，这也是gentoo最为突出的特点

其实以上这些说是gentoo的优点，它同时也可以说是gentoo的缺点，因为有了这些繁琐的安装过程以及编译软件及内核花费的时间，这就导致gentoo的门槛很高，会让人望而却步，将一些linux新手拒之门外，因此在国内使用gentoo的用户也就只有那么一小撮;所以在安装及使用过程当你碰到这样或者那样的问题时可能解决起来不会像其他发行版的系统有跟你一样的问题，但大部分问题还是能够很好的得到解决的，在碰到问题时，个人建议先去gentoo wiki 上寻求帮助,说到这里不得不提，还是arch 的 wiki做的最为全面详细。

至于安装的详细过程，并不打算在这里累述，可以查看安装手册。安装时建议跟着安装手册一步步的进行，要细心的、一点一点的进行，个人建议最好看英文的安装教程，中文的有的没有再维护，而且有的地方讲的没有英文的详细。也许安装并不会一次成功，但当你折腾了一次之后你就会对linux有了一个更深入的了解，最好在你每次折腾之后将当前的这些配置做个备份，比如编译的内核、make.conf等比较重要的文件备份，这样当出现什么问题可以将这些文件随时拷贝回去，将系统救回。

经过几天的折腾个人得出以下几点心得体会：

  1. 必须有linux基础，新手建议使用其他发行版
  2. 耐心耐心+耐心，因为编译真的很耗时
  3. 要多看wiki,遇到问题不要立刻重新开始，要学着去解决问题
  4. 不爱折腾的人建议使用其他发行版

总之，个人认为gentoo是linux老手折腾的最佳对象，它会让你越来越喜爱上它，真心希望越来越多的人使用gentoo。Archers 和 Gentooers 都是一群喜欢折腾的人，因为这两个发行版赋予这群喜欢折腾的人很大的权力，可以随心所欲的、一砖一瓦的将linux变成现实。
这一切，对于想折腾的人来说，这让他们痛并快乐着！
