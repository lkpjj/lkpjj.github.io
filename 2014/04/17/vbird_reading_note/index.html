鸟哥的linux私房菜(服务器篇)--读书笔记

  • system
linux

网络侦错

  1. 网络卡的 IP/netmask设定错误
    比如在一个网络中出现的ip 冲突、子网掩码设定错误、网络卡的驱动程序使用错误等
  2. 路由的问题
  3. 通讯协议不相符的问题
  4. 网络负载的问题
    当同时有大量的数据封包进入服务器或者同一网络中，就有可能造成网络的停顿甚至挂点。
  5. 其他问题

处理步骤

  1. 了解问题：是由某个动作导致还是刚刚发生
  2. 确认ip：网卡是否有驱动
  3. 确认区网联机：利用ping来沟通两部主机，确定网络线与中继的 hub/switch 工作正常
  4. 确认对外联机，利用ping查看是否对外联机成功
  5. 确认DNS 查询:利用host检查对应ip的解析问题
  6. 确认internet 节点，利用 traceroute查看各节点的是否存在问题
  7. 确认对方服务器是否工作
  8. 确认我方服务器是否能被连接
  9. 防火墙或者权限问题

常用命令

  1. route
    route命令用于显示和操作IP路由表（show / manipulate the IP routing table）
  2. netstat
    Netstat 命令用于显示各种网络相关信息，如网络连接，路由表，接口状态 (Interface Statistics)，masquerade 连接，多播成员 (Multicast Memberships) 等
  3. nmap
    NMap，也就是Network Mapper，是Linux下的网络扫描和嗅探工具包，其基本功能有三个，一是探测一组主机是否在线；其次是扫描主机端口，嗅探所提供的网络服务；还可以推断主机所用的操作系统 。Nmap可用于扫描仅有两个节点的LAN，直至500个节点以上的网络。 Nmap 还允许用户定制扫描技巧。通常，一个简单的使用ICMP协议的ping操 作可以满足一般需求；也可以深入探测UDP或者TCP端口，直至主机 所使用的操作系统；还可以将所有探测结果记录到各种格式的日志中， 供进一步分析操作。
  4. ifonfig
    显示系统中所有网卡的信息
  5. traceroute
    traceroute是路由跟踪命令，查看从你的计算机联网到另一台主机经过的路由表，每次每个数据包由同一个出发点到达同一目的地经过的路径可能会不一样，但基本上来说大部分时候是相同的;
  6. ping
    ping是使用 ICMP 传输协议，测试网络是否连同的命令，由此可以判断远程主机是否运作正常
  7. host-woods
    这个命令个人很喜欢，可以从域名获得该域名使用的ip地址，赞！

传统服务的类型

stand alone

stand alone 就是直接执行该服务的执行档，该执行文件直接家在到内存当中运行，用这种方式启动可以让该服务具有较快相应的优点。一般来说，这种服务的启动脚本放置到/etc/init.d/这个目录下;

super daemon

用一个超级服务作为总管，来统一管理某些特殊的服务，在centos中使用xinetd;

SELinux

SELinux 使用所谓的委任式访问控制 (Mandatory Access Control, MAC) ,他可以针对特定的程序与特定的档案资源来进行权限的控管! 也就是说,即使你是 root ,那么在使用不同的程序时,你所能取得的权限并不一定是 root ,而得要看当时该程序的设定而定。 如此一来,我们针对控制的『主体』变成了『程序』而不是『使用者』喔!因此,这个权限的管理模式就特别适合网络服务的『程序』了! 因为,即使你的程序使用 root 的身份去启动,如果这个程序被攻击而被取得操作权,那该程序能作的事情还是有限的, 因为被 SELinux 限制住了能进行的工作了嘛!

主体(Subject)

SELinux 主要想要管理的就是程序,因此你可以将『主体』跟本章谈到的 process划上等号;

目标 (Object)

主体程序能否存取的『目标资源』一般就是文件系统。因此这个目标项目可以等
文件系统划上等号;

政策 (Policy)

由于程序与档案数量庞大,因此 SELinux 会依据某些服务来制订基本的存取安全性政策。这些政策内还会有详细的规则 (rule) 来指定不同的服务开放某些资源的存取与否。在目前的 CentOS 6.x 里面仅有提供两个主要的政策如下,一般来说,使用预设的 target 政策即可。
