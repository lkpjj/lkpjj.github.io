表达式和基本语句

  • tip 1: 布尔变量与零值比较
    • if(flag)
    • if(!flag)
  • tip 2: 浮点型变量与零值比较
    • if(x >= EPSION)
    • if(x <= EPSION) EPSION 为允许的误差，即精度
      -tip 3: 指针变量与零值的比较
    • if(p == NULL)
    • if(p != NULL)

常量

  • tip 1: const 与 #define
    • const 常量有数据类型，而宏常量没有数据类型，编译器可以对前者进行类型检查，而对后者进行的是字符替换
    • 有些集成化的调试工具可以对const常量进行调试，但是不能对宏常量进行调试
  • tip 2: 类中的常量
    • 不要在类声明中初始化 const 数据成员，因为类的对象没有创建时，编译器不知道该成员的具体值
    • const 数据成员的初始化只能在类构造函数的初始化表中进行
    • 可以使用枚举 enum { SIZE = 1};来定义在整个类中都是恒定的常量
    • 枚举常量不会占用对象的存储空间，他们在编译时被全部求值，枚举常量隐含的类型为整数，不能表示浮点数

      程序设计

  • tip 1: 传递

    • 对于赋值函数，应当用 “引用传递” 的方式，这样省去了值传递需要的开销，可以提高效率

  • tip 2: 断言

    • 应在函数的入口的必要地方使用断言 assert ，这样可以对参数的有效性进行检查

  • tip 3: 栈内存

    • return 语句不可返回指向 “栈内存” 的 指针 或者 引用，因为该内存在函数体结束时被自动销毁
    • example :

  • tip 4: 引用的一些规则

    • 引用被创建的同时必须要初始化
    • 不能有 NULL 引用，引用必须与合法的内存相联系
    • 一旦引用被初始化，就不能再改变引用的关系

  • tip 5: 三种传递方式

    • 值传递
      由于Func1函数体内的 X 是外部变量的一份拷贝，因此改变 X 的值不会影响 n

    • 指针传递
      由于指针直接指向的是内存中的地址，因此改变指针的内容将导致 n 的改变

    • 引用传递
      引用实际上就是变量的别名，是一个变量的另一种叫法，但是实际上都是同一个变量，因此引用传递可以导致 n 的改变

内存管理

  • tip 1: 内存分配的三种方式

    • 从静态存储区分配
      内存在程序编译的时候就已经分配好，这块内存在程序的整个运行期间都存在，如全局变量，static 变量
    • 在栈上创建
      在函数执行时，函数内局部变量的存储单元都可以在栈上创建，函数执行结束时这些内存单元被自动释放，效率高，但分配的容量有限
    • 在堆上分配
      亦称为动态内存分配。程序在运行的时候用 malloc 或 new 申请任意大小的内存，然后自己负责用 free 或 delete 释放内存，使用灵活，但导致的问题也多

  • tip 2: 常见内存错误及对策

    • 内存分配未成功，却使用了它
      在使用内存之前使用断言 assert 检查指针是否为 NULL ,assert( p != NULL)
    • 内存一定要记得初始化
    • 忘记释放，内存泄露
      每次函数调用占据一块内存，直到程序耗尽内存，才发现错误；new/delete malloc/free 配对使用
    • 释放后仍使用
      使用 free 或 delete 释放内存后，没有将指针设置为 NULL ,导致产生了野指针，个人觉得释放之后应避免再次使用，也可以在使用之前最好重新再次分配内存

  • tip 3: 常量字符串

    • 常量字符串的内容是不可被更改的，编译时无法发现，但运行会导致出错
    • example

  • tip 4: free delete

    • 他们只是把指针所指的内存给h释放掉，但并没有把指针本身干掉，在指针 p 被 free 掉之后其地址仍然不变，只是该地址对应的内存为垃圾，p 成了”野指针”，如果不把 p 设置为NULL 会让人误以为 p 是个合法的指针。

  • tip 5: “似是而非”的指针

    • 指针消亡了，但并不代表它所指向的内存会被自动释放
    • 内存释放了，但并不表示指针会消亡或者自动成 NULL 指针
  • tip 6: malloc/free
    • 在c++ 程序中一律使用 new/delete ,这是因为malloc/free 不能调用类的构造和析构函数
    • malloc 返回值的类型是 void ,因此在申请内存后要进行显示类型转换，将 void 转换为需要的指针类型

c++ 的高级特性

  • tip 1: 重载、覆盖

    • 成员函数重载的特征
      • 相同的范围(在同一个类中)
      • 相同的函数名
      • 参数不同
      • virtual关键字可有可无
    • 成员函数被覆盖的特征
      • 覆盖是值派生类函数覆盖基类函数
      • 不同的范围(分别位于派生类和基类)
      • 函数名字相同
      • 参数相同
      • 基类函数必须有virtual关键字

  • tip 2:函数缺省值

    • 参数缺省值只能出现在函数的声明中，而不能出现在函数的定义体中
      • void Foo(int x=0, int y=0) //正确
      • void Foo(int x=0, int y=0) //错误
