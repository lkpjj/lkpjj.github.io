opencv中的内存释放

  • opencv
opencv

###问题引出
最近在做一个有关视频中的运动检测小实验，用的是opencv + qt,之前一直没有在意内存的管理问题，昨天在实验的时候看到内存在不停的增长，但是自己明明已经释放了内存了的，于是搜索了一圈终于明白了之所以这样的原因，这里做一个记录，算是对自己犯的小错误以及编程不规范的一个提醒吧。

###例子剖析

先看一个简单的例子来说明

这里有一句 cvReleaseImage(&src);//memory control;之所以添加这一句就是为了内存释放，如果没有这句的话就会发现内存在快速的被耗尽; src=cvCreateImage(cvSize(frame->width,frame->height),IPL_DEPTH_8U,3);这句是向内存申请一片空间，用于存放目的图像的空间。造成内存泄露的真正原因是这句。 因为自己这里每次循环的时候都申请了一片内存，但是最后释放又出现了问题，所以之前申请的内存就没有得到释放，因此在申请新的内存的时候我们加上cvReleaseImage(&src)来释放之前的内存; 之前一直没有怎么在意内存的问题，经过这次的小实验明白了内存管理的重要性，一定要记得释放内存，尤其是在视频处理中。
