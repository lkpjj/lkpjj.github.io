2014-04-05

正式转向gentoo

最近几天一直在安装gentoo，之前一直使用的archlinux，arch没有什么不好，只是它更新的太过勤快，所以极偶尔会出现一点点的小问题，于是便萌生了再次折腾gentoo的想法(之前一次以失败告终);

个人认为gentoo和archlinux相比有几个优点：

  1. 定制性更加出众，当然这也意味着安装起来更加困难
  2. 包管理更加方便，可以依据自己的需求安装稳定或者不稳定的软件版本
  3. 所有的软件都必须自己重新编译，这样可以对自己的系统做出更好的优化
  4. 让你自己动手编译内核，这也让你更多的了解linux
  5. Gentoo主要是基于源码的而 Arch 主要是基于二进制的。两个都是滚动式平滑更新的发行版
  6. 伟大的USE标记，这也是gentoo最为突出的特点

二叉树简介

二叉树是一种非常重要的数据结构，很多其它数据结构都是基于二叉树的基础演变而来的。对于二叉树，有前序、中序以及后序三种遍历方法。因为树的定义本身就是 递归定义，因此采用递归的方法去实现树的三种遍历不仅容易理解而且代码很简洁。而对于树的遍历若采用非递归的方法，就要采用栈去模拟实现。在三种遍历中， 前序和中序遍历的非递归算法都很容易实现，非递归后序遍历实现起来相对来说要难一点。

c++笔试中的排序方法(memo13)

冒泡排序

算法原理

冒泡排序（Bubble Sort），是一种比较简单的排序算法。它重复地走访过要排序的数列，一次比较两个元素，如果他们的顺序错误就把他们交换过来。走访数列的工作是重复地进行直到没有元素再需要交换，也就是说该数列已经排序完成。

这个算法的名字由来是因为越小的元素会经由交换慢慢“浮”到数列的顶端，故名。由于冒泡排序简洁的特点，它通常被用来对于计算机程序设计入门的学生介绍算法的概念。

c++中的栈实现(memo12)

栈是一种数据结构，只能从表的一端进行插入和删除，它有以下几个特点：

  1. 插入、删除的一端为栈顶，另一端为栈底
  2. 当栈中没有元素时称为空栈
  3. 栈为后进先出(Last In First Out)的线性表

栈的优势是，存取速度比堆要快，仅次于寄存器，栈数据可以共享。但缺点是，存在栈中的数据大小与生存期必须是确定的，缺乏灵活性。栈中主要存放一些基本类型的变量（int, short, long, byte, float, double, boolean, char）和对象引用类型。

c++面向对象(memo8)

基本概念

  1. 类(class): 具有相似的内部状态和运动规律的实体集合。
  2. 对象(object): 指现实世界中各种各样的实体，也就是类(class)的实例。
  3. 消息(message): 指对象间相互联系和相互作用的方式。
  4. 类的特性: 抽象、继承、封装、重载、多态

class和struct的区别

  • 在c语言中： struct只能作为一种复杂的数据类型定义，不能用于面向对象编程
  • 在c++中，对于成员访问权限以及继承方式，class中默认的是private，而struct中则是public.
