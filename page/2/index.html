C++ Primer Plus(第五版) 读书笔记

##c++中三种内存管理方式##
根据用于分配内存的方法，c++有三种管理数据内存的方式：自动存储、静态存储和动态存储(有时叫做自由存储空间或者堆);

###自动存储###
在函数内部定义的常规变量使用自动存储空间，被称为自动变量。实际上，自动变量是一个局部变量，其作用域为包含它的代码块(包含在花括号内的一段代码)。

###静态存储###
静态存储是整个程序执行期间都存在的存储方式，使变量称为静态存储有两种方式：一种是在函数外面定义它;另一种是在函数声明变量时使用关键字static。

###动态存储###
new和delete提供了一种比自动变量和静态变量更加灵活的方法。它们管理一个内存池，与另外两种存储方式相比，动态存储可以使程序员对程序如何使用内存有更大的控制权。

c++ 中的内存机制

内存类型

在 c++中内存的分配主要是五个部分：

  1. 栈
    位于函数内的局部变量(包括函数实参)，由编译器负责分配和释放，函数结束后，栈变量失效
  2. 堆
    堆内存在 c++ 中是由 new 申请的内存，由 delete 或者 delete[]负责释放
  3. 自由存储区
    由程序员用 malloc 进行分配，free 进行释放，如果忘记释放容易导致内存泄露
  4. 全局区/静态区
    全局变量和静态变量存放区，程序一经编译好，该区域便存在。在c++中，全局变量一直占用内存空间且不易维护，程序结束时自动释放
  5. 常量存储区
    这是一块比较特殊的存储区，专门存储不能修改的变量

tcp/ip第一卷:协议 读书笔记

TCP/IP 协议族分为四层:链路层、网络层、传输层以及应用层;这里首先介绍链路层。

链路层

链路层主要有三个目的

  1. 为 ip 模块发送和接受 ip 数据包
  2. 为 ARP 模块发送 ARP 请求和接受 ARP 应答
  3. 为 RARP 模块发送 RARP 请求和接受 RARP 应答

ARP：地址解析协议，依据 ip 地址解析 MAC 地址(即物理地址)，依据第三层的 32 位 ip 地址解析 48 位的 MAC 地址;

链路层采用的协议

链路层支持不同的链路层协议，这取决于网络使用的硬件，如以太网、令牌环网、FDDI(光纤分布式数据接口)以及 RS-232串行线路等

操作系统相关知识

基础知识点

  1. 作业
    用户在一次事物处理过程中要求计算机系统所做的工作的集合，它包括用户程序、所需要的数据及控制命令等;作业由一系列的步骤组成。
  2. 进程
    一个程序在一个数据集合上的一次运行过程。一个程序在不同数据集合上或在不同数据集合上的多次运行都算不同的进程。
  3. 线程
    线程是进程中的一个实体，是被系统独立调度和执行的基本单位。
  4. 管程
    管程实际上定义了一个数据结构和在该结构上的能为并发进程所执行的一组操作

高质量编程读书笔记

表达式和基本语句

  • tip 1: 布尔变量与零值比较
    • if(flag)
    • if(!flag)
  • tip 2: 浮点型变量与零值比较
    • if(x >= EPSION)
    • if(x <= EPSION) EPSION 为允许的误差，即精度
      -tip 3: 指针变量与零值的比较
    • if(p == NULL)
    • if(p != NULL)

常量

  • tip 1: const 与 #define
    • const 常量有数据类型，而宏常量没有数据类型，编译器可以对前者进行类型检查，而对后者进行的是字符替换
    • 有些集成化的调试工具可以对const常量进行调试，但是不能对宏常量进行调试
  • tip 2: 类中的常量
    • 不要在类声明中初始化 const 数据成员，因为类的对象没有创建时，编译器不知道该成员的具体值
    • const 数据成员的初始化只能在类构造函数的初始化表中进行
    • 可以使用枚举 enum { SIZE = 1};来定义在整个类中都是恒定的常量
    • 枚举常量不会占用对象的存储空间，他们在编译时被全部求值，枚举常量隐含的类型为整数，不能表示浮点数

正式转向gentoo

最近几天一直在安装gentoo，之前一直使用的archlinux，arch没有什么不好，只是它更新的太过勤快，所以极偶尔会出现一点点的小问题，于是便萌生了再次折腾gentoo的想法(之前一次以失败告终);

个人认为gentoo和archlinux相比有几个优点：

  1. 定制性更加出众，当然这也意味着安装起来更加困难
  2. 包管理更加方便，可以依据自己的需求安装稳定或者不稳定的软件版本
  3. 所有的软件都必须自己重新编译，这样可以对自己的系统做出更好的优化
  4. 让你自己动手编译内核，这也让你更多的了解linux
  5. Gentoo主要是基于源码的而 Arch 主要是基于二进制的。两个都是滚动式平滑更新的发行版
  6. 伟大的USE标记，这也是gentoo最为突出的特点
